# Software testing
测试是为了“破坏”
###### 好的测试？
1. 能发现错误
2. 不冗余
3. 有最佳特性
4. 别太复杂也别太简单
## 测试等级
回归测试分为三类
![image.png](https://s2.loli.net/2024/05/25/bOkzoa51PhlgEtR.png)
## 一些概念
1. 静态和动态测试：静态只能发现一些语法错误或者死循环（IDE的代码检查），而动态测试检查逻辑上的问题，通过结果来执行
2. 测试和Debug：发现错误和消除错误
3. **白盒测试**对内部代码结构进行测试，**黑盒测试**对程序表现的东西测试，比如输入得到什么输出
